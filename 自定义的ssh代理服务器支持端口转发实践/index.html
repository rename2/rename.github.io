端口转发简介#

端口转发(Port Forwarding)是SSH中的一种机制，可以通过已经建立的SSH连接转发其他端口和应用的网络流量。SSH通道本身提供了加密和解密功能，保障了传输的安全性，所以端口转发机制也被称为隧道(tunneling)。
端口转发的使用方法有3种。

本地端口转发#

本地端口转发将本地端口收到的请求发给服务器，服务器连接对应的目标和端口（目标地址可以是其他服务器）。这种方式常用于打破通信限制，从外部连接内网的服务（服务器正向代理流量）。
本地端口转发的命令格式是ssh -L [local_bind_address:]local_port:host:hostport remote_host。
如果想访问host1上的8080端口，可以执行ssh -L 80:localhost:8080 host1。然后访问本地的80端口、收到host1 8080端口的响应。
如果想通过host1的去访问host2，可以用ssh -L 80:host2:8080 host1。
如果想一个命令开启多个端口的转发，指定多个转发参数-L xxx -L xxx就可以做到。

动态端口转发#

本地端口转发也可以是动态的，将本地端口的请求发给多个目标地址。
动态端口转发的命令格式是ssh -D local_port host1。
比如ssh -D 2001 host1开放了本地的2001端口，之后可以通过curl -x socks5://localhost:2001 localhost:8080访问host1的8080端口，或者通过curl -x socks5://localhost:2001 host2:5001访问host2的5001端口。

远程端口转发#

远程端口转发在远程服务器绑定一个端口，将这个端口收到的请求转回本地，本地连接对应的目标和端口。这种方式常用于内网穿透和打开后门（发起SSH请求的本地反向代理流量）。
远程端口转发的命令格式是ssh -R [remote_bind_address:]remote_port:host:hostport host1。
ssh -R 8888:localhost:8080 host1会在host1上绑定8888端口，将收到的请求转回本地8080端口。
ssh -R localhost:8888:host2:8080 host1会在host1上绑定8888端口，将收到的请求转回本地，本地访问host2的8080端口。
远程服务器上绑定的端口默认只有该机器上可以访问，如果需要其他机器也能访问，要修改远程服务器上的sshd配置为GatewayPorts yes或GatewayPorts clientspecified（客户端指定）。
同样地，指定多个转发参数-R xxx -R xxx可以开放多个端口。

端口转发原理#

SSH连接中会发送两种消息：channel和request。
channel表示单个服务的处理，如session、direct-tcpip（本地转发）和forwarded-tcpip（远程转发）。
request表示向另一方请求的操作，具体有两种：全局和channel内部。常见的全局request有表示保持连接的keepalive@openssh.com和不再打开更多会话的no-more-sessions@openssh.com。channel内部的request包括session channel中的shell、subsystem、windows-change等。request可自定义，只要SSH server有实现对应的处理。

SSH连接支持同时执行多个操作，如交互式执行命令、SFTP上传下载文件和端口转发。
使用端口转发时，每次访问都会发送一个direct-tcpip channel请求，其中附带目标地址。
也就是说，通过ssh -L 80:localhost:8080 host1启用端口转发后，每次curl localhost:80都会触发一个channel请求。
对于远程端口转发，建立连接后会发送tcpip-forward channel请求以预先绑定机器上的端口。本地端口转发没有这个操作。

代理服务器实现端口转发#

本文的场景是实现一个SSH代理服务器，提供权限校验、高危命令拦截、操作审计等功能。
ProxyCommand可以配置代理服务器。SSH连接代理服务器同样是通过direct-tcpip channel，其中指定目标地址。
本地端口转发请求是触发式的，只需要在收到direct-tcpip channel后把请求转发到目标地址。
简化的端口转发实现如下。整体使用了开源库 github.com/gliderlabs/ssh 搭建 SSH server。 DirectTCPIPHandler处理direct-tcpip channel，是所有请求的入口。LocalPortForwardingHandler处理本地端口请求，只需要在目标机器上访问指定的地址，然后转发给用户。

import (
    "errors"
    "fmt"
    "io"
    "net"
    "strconv"
    "time"

    "github.com/gliderlabs/ssh"
    gossh "golang.org/x/crypto/ssh"
)

func main() {
    signer, err := gossh.ParsePrivateKey([]byte(privateKey))
    if err != nil {
        fmt.Println(err)
        return
    }

    addr := "0.0.0.0:2222"
    sshServer := &ssh.Server{
        Addr:        addr,
        HostSigners: []ssh.Signer{signer},
        Handler:     SessionHandler,
        ChannelHandlers: map[string]ssh.ChannelHandler{
            "session":      ssh.DefaultSessionHandler,
            "direct-tcpip": DirectTCPIPHandler,
        },
    }

    listener, err := net.Listen("tcp", addr)
    if err != nil {
        fmt.Println(err)
        return
    }

    if err := sshServer.Serve(listener); err != nil {
        fmt.Println("serve:", err)
    }
}

func DirectTCPIPHandler(srv *ssh.Server, conn *gossh.ServerConn, newChan gossh.NewChannel, ctx ssh.Context) {
    defer conn.Close()

    d := localForwardChannelData{}
    if err := gossh.Unmarshal(newChan.ExtraData(), &d); err != nil {
        _ = newChan.Reject(gossh.ConnectionFailed, fmt.Sprintf("error parsing forward data: %s", err.Error()))
        return
    }

    signer, err := gossh.ParsePrivateKey([]byte(destPrivateKey))
    if err != nil {
        _ = newChan.Reject(gossh.ConnectionFailed, fmt.Sprintf("error parsing dest private key: %s", err.Error()))
        return
    }

    dest := net.JoinHostPort(d.DestAddr, strconv.FormatInt(int64(d.DestPort), 10))
    config := &gossh.ClientConfig{
        User:            ctx.User(),
        Auth:            []gossh.AuthMethod{gossh.PublicKeys(signer)},
        HostKeyCallback: gossh.InsecureIgnoreHostKey(),
        Timeout:         10 * time.Second,
    }
    client, err := gossh.Dial("tcp", dest, config)
    if err != nil {
        _ = newChan.Reject(gossh.ConnectionFailed, err.Error())
        return
    }
    ctx.SetValue(ContextKeySSHClient, client)

    proxyCh, reqs, err := newChan.Accept()
    if err != nil {
        fmt.Println("accept:", err)
        return
    }
    defer proxyCh.Close()
    go gossh.DiscardRequests(reqs)

    proxyConn := chanConn{
        Channel: proxyCh,
        laddr: &net.TCPAddr{
            IP:   net.ParseIP(d.OriginAddr),
            Port: int(d.OriginPort),
        },
        raddr: &net.TCPAddr{
            IP:   net.ParseIP(d.DestAddr),
            Port: int(d.DestPort),
        },
    }
    sshConn, chans, proxyConnReqs, err := gossh.NewServerConn(&proxyConn, getServerConfig(srv, ctx))
    if err != nil {
        fmt.Println("create proxy connection:", err)
        return
    }
    defer sshConn.Close()
    go gossh.DiscardRequests(proxyConnReqs)

    for ch := range chans {
        handler := srv.ChannelHandlers[ch.ChannelType()]
        if handler == nil {
            handler = srv.ChannelHandlers["default"]
        }
        if ch.ChannelType() == "direct-tcpip" {
            handler = LocalPortForwardingHandler
        }
        if handler == nil {
            ch.Reject(gossh.UnknownChannelType, "unsupported channel type")
            continue
        }
        go handler(srv, sshConn, ch, ctx)
    }
}

func LocalPortForwardingHandler(srv *ssh.Server, conn *gossh.ServerConn, newChan gossh.NewChannel, ctx ssh.Context) {
    d := localForwardChannelData{}
    err := gossh.Unmarshal(newChan.ExtraData(), &d)
    if err != nil {
        _ = newChan.Reject(gossh.ConnectionFailed, "error parsing forward data: "+err.Error())
        return
    }

    client, ok := ctx.Value(ContextKeySSHClient).(*gossh.Client)
    if !ok {
        _ = newChan.Reject(gossh.ConnectionFailed, "error getting ssh client: "+err.Error())
        return
    }

    dest := net.JoinHostPort(d.DestAddr, strconv.FormatInt(int64(d.DestPort), 10))
    dconn, err := client.DialContext(ctx, "tcp", dest)
    if err != nil {
        newChan.Reject(gossh.ConnectionFailed, err.Error())
        return
    }

    ch, reqs, err := newChan.Accept()
    if err != nil {
        dconn.Close()
        return
    }
    go gossh.DiscardRequests(reqs)

    go func() {
        defer ch.Close()
        defer dconn.Close()
        io.Copy(ch, dconn)
    }()
    go func() {
        defer ch.Close()
        defer dconn.Close()
        io.Copy(dconn, ch)
    }()
}

// Same as localForwardChannelData in github.com/gliderlabs/ssh.
type localForwardChannelData struct {
    DestAddr string
    DestPort uint32

    OriginAddr string
    OriginPort uint32
}

// Same as chanConn in golang.org/x/crypto/ssh.
type chanConn struct {
    gossh.Channel
    laddr, raddr net.Addr
}

func (t *chanConn) LocalAddr() net.Addr {
    return t.laddr
}

func (t *chanConn) RemoteAddr() net.Addr {
    return t.raddr
}

func (t *chanConn) SetDeadline(deadline time.Time) error {
    if err := t.SetReadDeadline(deadline); err != nil {
        return err
    }
    return t.SetWriteDeadline(deadline)
}

func (t *chanConn) SetReadDeadline(deadline time.Time) error {
    return errors.New("deadline not supported")
}

func (t *chanConn) SetWriteDeadline(deadline time.Time) error {
    return errors.New("deadline not supported")
}

// Reference: github.com/gliderlabs/ssh/server.go config()
func getServerConfig(srv *ssh.Server, ctx ssh.Context) *gossh.ServerConfig {
    var config = &gossh.ServerConfig{}
    for _, signer := range srv.HostSigners {
        config.AddHostKey(signer)
    }
    if srv.PasswordHandler == nil && srv.PublicKeyHandler == nil && srv.KeyboardInteractiveHandler == nil {
        config.NoClientAuth = true
    }
    if srv.Version != "" {
        config.ServerVersion = "SSH-2.0-" + srv.Version
    }
    if srv.PublicKeyHandler != nil {
        config.PublicKeyCallback = func(conn gossh.ConnMetadata, key gossh.PublicKey) (*gossh.Permissions, error) {
            if ok := srv.PublicKeyHandler(ctx, key); !ok {
                return ctx.Permissions().Permissions, fmt.Errorf("permission denied")
            }
            ctx.SetValue(ssh.ContextKeyPublicKey, key)
            return ctx.Permissions().Permissions, nil
        }
    }
    return config
}

const (
    ContextKeySSHClient contextKey = iota
)

连接代理服务器#

命令行#

命令行使用代理服务器进行端口转发时，以下参数很有用。

  • -N 不开启session，只执行端口转发。
  • -f 把SSH连接放到后台，就不再需要单独开启一个终端跑SSH。可以搭配-M（启用连接复用）和-S参数（连接复用的socket地址）使用。用ssh … -O exit停止端口转发。
  • ExitOnForwardFailure 出现转发错误就退出连接。

VSCode Remote-SSH#

Remote SSH会建立两个SSH连接来打开远程窗口：第一个连接用于安装和启动VS Code 服务器，第二个连接用于连接已启动的 VS Code端口。其中端口转发是建立的长连接，在退出远程开发时才会关闭。
Remote SSH 连接我们的代理服务器只需要配置SSH配置文件，配置文件内容类似下面的内容。 

Host *
  HostName %h
  IdentityFile ~/id_rsa
  IdentitiesOnly yes
  ProxyCommand ssh -i "~/id_rsa" -o "IdentitiesOnly=yes" -p 2222 -F /dev/null {proxy_address} -W %h:%p
  SendEnv LANG LC_*

同时可以启用窗口间的连接共享，提升使用体验。 连接后可以在输出里看到详细日志。
